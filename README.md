# decodelabs-internship-week2
This project implements a K-Nearest Neighbors (KNN) classification model to identify iris flower species based on their physical measurements, using the classic Iris dataset. The dataset contains 150 samples across three species — Setosa, Versicolor, and Virginica — each described by four numerical features: sepal length, sepal width, petal length, and petal width. The project follows a structured Input-Process-Output (IPO) pipeline that mirrors real-world machine learning workflows. In the Input stage, the dataset is loaded and explored to understand its shape and class distribution. The Process stage begins by splitting the data into training (80%) and testing (20%) subsets using a stratified split, which ensures that the proportion of each species remains balanced across both sets. Before training, the features are standardized using StandardScaler so that all measurements share a common scale (mean of 0 and variance of 1), preventing any single feature from disproportionately influencing the distance-based KNN algorithm. The model itself is a KNeighborsClassifier that predicts a flower's species by examining the five nearest data points in the training set and assigning the most common label among them. In the Output stage, the model's performance is evaluated using multiple metrics rather than relying on accuracy alone: a confusion matrix highlights where misclassifications occur, while precision, recall, and F1-score provide a more complete picture of how well the model distinguishes between the three species. As a bonus step, the script also performs a simple hyperparameter search, testing K values from 1 to 20 to identify which number of neighbors yields the highest test accuracy, demonstrating how model performance can be fine-tuned. Overall, this project serves as a practical, beginner-friendly introduction to supervised classification, feature scaling, and model evaluation techniques commonly used in applied machine learning tasks.
